## 根据实际部署反馈优化 Skill
基于实际部署测试发现的两个问题，更新了 SKILL.md 和 deploy.sh：
1. **远程构建超时问题**：EdgeOne 远程构建时 `mirrors.tencent.com` 连接超时
   - SKILL.md Step 3 重写为"优先本地构建"策略，先 `npm run build` 再部署产物
   - deploy.sh 添加超时错误检测，自动回退到本地构建再重试部署
2. **项目数量超限（40个上限）**：`Pages project exceeds 40 limit`
   - SKILL.md Troubleshooting 添加完整处理流程：
     - Option 1: 引导用户去控制台 https://edgeone.ai/pages 手动删除
     - Option 2: 用户提供 api_token + project_id，通过 DeletePagesProject API 删除
   - deploy.sh 添加 `--delete` 模式，可通过脚本调用 API 删除项目
   - deploy.sh 部署失败时自动检测错误类型并给出针对性建议

## 第三轮优化（3个问题修复）
1. **API Token 控制台链接**：所有提到 API Token 的地方都加上了创建链接
   - 国内站: https://console.cloud.tencent.com/edgeone/pages?tab=settings
   - 国际站: https://console.tencentcloud.com/edgeone/pages?tab=settings
2. **部署成功输出完整 URL**：
   - 要求 AI 展示完整的 `EDGEONE_DEPLOY_URL`（带 eo_token 和 eo_time 参数）
   - 提醒预览 URL 有效期只有 3 小时
   - 引导用户去控制台绑定自定义域名获取永久 URL
3. **修正部署流程**：
   - 旧流程：先本地 `npm run build` 再部署产物
   - 新流程：先 `edgeone pages deploy`（远程构建）→ 失败则 `edgeone pages build`（本地构建）→ `edgeone pages deploy .edgeone`
   - deploy.sh 同步修正为新流程
